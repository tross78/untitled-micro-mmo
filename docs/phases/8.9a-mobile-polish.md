# Phase 8.9a — Mobile Layout and Interaction Polish

Sourced from the gameplay audit (2026-05-15). Items below are confirmed issues deferred from the 8.78 fix pass.

## Input / Controls

- **Adaptive UI by input mode** — `lastInputMode` is tracked but HUD buttons and menu labels stay static. Touch/gamepad users still see keyboard-centric affordances. Fix: adapt all visible prompts, button legends, and footer copy by mode (not just dialogue hints).
- **Touch hold/drag-to-repeat movement** — swipe gives one discrete step. Holding a swipe or dragging should emit continuous movement like held keyboard. Fix: add optional held virtual d-pad or drag-to-repeat on touch canvas.
- **Gamepad drift during UI** — analog stick polling continues during menus/dialogue; input is drained, but if a stick is deflected when the menu closes, movement fires immediately. Fix: pause stick movement polling while UI owns input, or clear active movement actions on menu/dialogue open.
- **Held key resume after dialogue** — held keyboard movement during dialogue/menu resumes after close without a fresh keypress, because repeat timers keep firing every 110ms. Fix: clear held movement repeat state when dialogue/menu opens.
- **forestFights exhaustion affordance** — attack button remains visible when hunts are depleted; tapping it fails with a log message. Fix: disable or annotate attack affordances when `forestFights === 0`.
- **Mobile browser back interception** — in-game back flow exists, but a native back swipe/button can leave the page. Fix: intercept `popstate`/`history` where appropriate or show a "leave game?" prompt on session exit.
- **Confirm dialogs for destructive taps** — buying, selling, or spending rare items has no confirm step. A fat-finger can spend immediately. Fix: add confirm-on-purchase/sale for mobile or for expensive items.

## Visual / Feel

- **Collision bump feedback** — bump lasts ~3 frames at 60fps, barely perceptible on mobile. Fix: slow bumps slightly or pair with a stronger shake/sound pulse.
- **Idle animations** — player and NPCs are fully static when not moving; only tweening entities get walk pose. Fix: add a subtle idle bob/blink/sway.
- **Entity draw order** — entities are drawn in creation order, not y-order; shadows and bodies can overlap incorrectly. Fix: sort visible entities by world Y before drawing.
- **Threat level in HUD** — threat only appears in status/log surfaces, not the canvas top bar during play. Fix: surface threat in the top HUD with a normalised scale (e.g. ☠️3/5).
- **Legacy DOM status bar overflow** — the legacy strip concatenates room name + equipment + status icons into one unbounded center string. Fix: retire the legacy strip or add truncation/wrapping.

## Copy / UX

- **Locked quest row styling** — locked rows are only distinguished by `[locked]` text and disabled tint; easy to miss on small mobile panel. Fix: add iconography or a dedicated lock row style.
- **"No quests right now" dead copy** — gives no temporal clue. Fix: replace with the actual gate reason when known (e.g. "Complete Wolf Hunt first").
- **Touch footer copy** — `Esc/Back to exit` is meaningless on mobile. Fixed in 8.78 fix pass; verify the new copy ("Tap outside to back") is correct across menu types.
