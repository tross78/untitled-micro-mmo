# Phase 8.78 — Game-Readiness Pass: Core Loop Integrity

**Status:** COMPLETE  
**Depended on:** Phase 8.77 (World Vitality) — planned gate, implemented in parallel  
**Gates:** Phase 8.8 (Analytics) and Phase 8.9 (Private Alpha)

## Goal

Ensure the core player loop — questing, gathering, crafting, combat, and NPC dialog — is completable without softlocks, dead ends, or confusing gaps. Fix sloppy room presentation. Inspired by SNES Zelda (clear visual affordances), Stardew Valley (dedicated resource nodes per zone, satisfying gather loop), and Ultima Online (world items with real placement, varied acquisition paths).

## Delivered Slices

### 8.78a — Resource Nodes
- Added `'resource'` scatter type to `getScatteredContent` (`src/rules/world.js`)
- Log nodes placed in `forest_edge` (2–3) and `forest_depths` (3–5)
- Ore nodes placed in `cave` (2–3) and `mountain_pass` (1–2)
- Extended `handleInteract` (`movement-system.js`) to handle `type === 'resource'` with label map `{ log: 'wood', ore: 'iron' }`
- `log` and `ore` added to `SCENERY_SIZE_CLASSES.small` (`src/infra/graphics-constants.js`)
- Validate.js tracks resource nodes as item acquisition sources (same as flora)

### P0 — Depletion Gate (Shared-World foraging integrity)
- Added `gatheredNodes: { day: 0, nodes: new Set() }` to `localPlayer` in `src/state/store.js`
- Re-initialized after `loadLocalState` to prevent save-state bleed
- In `handleInteract` (`movement-system.js`): lazy-reset on day change, key `"roomId:x,y"`, one gather per node per in-game day per player
- Covers all scatter types (`flora` and `resource`) simultaneously
- Zero P2P coordination needed — depletion is per-player, consistent with the shared-world model

### 8.78b — Room Clutter Fixes
- `forest_depths` terrain density reduced 20 → 12; clutter trimmed to `['tree', 'shrub']`
- `smuggler_den` east wall gap closed (row y=4, x=8 was open with no exit there)
- `buildScatterBlockedTiles` (`world.js:235`) extended to parse `tiles` string-array format (`cave`, `watchtower`, etc.) — prevents scatter landing on `W`/`V` tiles
- Validate.js density cap: warns when `terrain.density > width × height × 0.15`
- Note: dungeon_cell and throne_room opening rows are intentional exit apertures, not gaps

### 8.78c — Quest Chain Completability
- `gather_wood` description: "Collect 5 wood bundles from the forest."
- `iron_supply` description: "Mine 3 iron ore bundles from the cave."
- Validate.js already enforces that every fetch quest target has ≥1 non-shop source (resource nodes now satisfy `wood` and `iron`)

### P3 — Quest Gate Visibility
- `getQuestRowsForNpc` in `canvas-menu.js` now shows locked quests with prerequisite labels: `"Mountain Trial [locked] — Requires: Cave Troll Bounty"`
- Fixes the silent militia-chain wall faced by barkeep players
- Logic: if giver matches and prerequisites exist but are unmet, show the quest as disabled with the unmet prerequisites named

### 8.78d — Shop Night Access
- Night shop restriction removed from merchant entirely (`canvas-menu.js`)
- Shops are now 24h vendors (Ultima Online style)
- Day/night still affects combat (enemy strength, wolf availability) — that is preserved

### P2 — Storm Fight Cost Surfaced (`combat-system.js`)
- Storm weather now logs "Storm: each fight costs N hunts. (X remaining)" before each fight on storm days
- When `forestFights <= 0`, shows time until UTC midnight reset: "Hunts reset in Xh Ym (UTC midnight)"
- Same reset time shown when storm exhausts fights prematurely

### P1 — Day/Night + Weather HUD Row (`ui-render-system.js`)
- Fourth row added to the top-bar stat panel: environment indicator
- Uses `getTimeOfDay()` and `worldState.weather`
- Icon: ☀ (day/clear), 🌙 (night/clear), ⛈ (storm), 🌫 (fog) — with night variants
- Color: `#ffe8a0` (day) or `#aac4ff` (night) for clear readability
- Communicates enemy strength modifier, wolf availability, merchant hours, and fight cost in one visual

### Step-onto Hint (`movement-system.js`)
- When player steps onto a flora or resource tile, emits a log hint: "Fallen log — press interact to gather." / "Wild herbs — press interact to forage."

### 8.78j — Daily Bounty Board (Endgame Loop, P4)
- Quest log now shows a "Daily Bounty" header using `worldState.bountyEnemy` (already derived daily from `deriveWorldState`)
- Post-capstone players see a daily target in `/quest list` and in the canvas quest menu
- Locked quests with partially-met prerequisites (player is in the chain but hasn't finished a gate) now appear in the global quest log with gate labels
