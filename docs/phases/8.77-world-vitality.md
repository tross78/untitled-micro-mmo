# Phase 8.77: World Vitality

**Status:** Upcoming (after Phase 8.76)  
**Goal:** Give the world observable personality — events that feel like news rather than system messages, and NPCs whose conversational range reflects their knowledge of world state and player history.

---

## 8.77a — Blaseball-Style Wild Events

Extend `EVENT_TABLE` in `src/rules/world.js` with a `WILD_EVENTS` array, gated on `threat >= 3`. Wild events break normal rules in legible ways and are announced as short news-headline strings. All effects are derived from `seededRNG + day` — no `Math.random()`, no new persistence.

| Event | Mechanical Effect | Narrative |
|---|---|---|
| `market_inversion` | Swap `scarcity` ↔ `surplus` arrays for the day | "The merchants are confused. Prices have inverted." |
| `cursed_loot` | Next item pickup applies -1 to a random stat for 50 rounds | "A witch's hex settles over the land." |
| `enemy_parley` | Combat disabled in one zone for 1 day | "The wolves have called a truce. Odd." |
| `merchant_amnesia` | All shop prices re-seeded for the day | "The traders forgot their ledgers." |
| `phantom_bounty` | Bounty target doesn't exist; guard pays anyway | "A ghost bounty appears on the board." |
| `ancient_reckoning` | All threat-level effects doubled for 1 day | "Something stirs beneath Hearthwick." |

Add a separate `world:wild` bus event (distinct from `world:event`) so UI can style wild events differently — larger toast, distinct colour. Add `WILD_EVENT` narrative strings to `src/content/data/constants.js`.

**Files:** `src/rules/world.js`, `src/network/simulation.js` (`_emitDayEvent()`), `src/state/eventbus.js` (new `world:wild` type), `src/main/events.js` (~line 497), `src/content/data/constants.js`

## 8.77b — NPC Dialogue 2.0: Intent-Driven + Semi-Procedural

Upgrade the Phase 8.75 contextual dialogue system with intent-driven line selection and lightweight stateful NPC memory.

**Intent layer:** each NPC corpus gets an `intents` array — a priority-ordered list of `{ intent, condition, lines }` entries. The selector walks the list and picks the highest-priority matching pool; base Markov generation is demoted to fallback when no intent matches. Conditions check `worldState`, `localPlayer.quests`, visit history, and active wild events.

```js
{ intent: 'post_wild_event', condition: ws => ws.wildEvent != null,
  lines: ["${wildEvent.narrative}", "Strange times. Best stay cautious."] }
```

**Stateful memory stubs:** visit count stored in `localPlayer.npcVisitCounts[npcId]`; switch to a `repeat_visit` pool after visit 2. No server-side persistence needed.

**Corpus target:** 40–60 lines per NPC across all pools. Each line should be in-world, non-generic, and reference at least one world-state value where plausible.

**Post-VPS option:** once the VPS relay is live (Phase 8.95e), the arbiter can proxy one LLM call per NPC per day, sign the result with the daily state, and broadcast it as the NPC's "today's word" — deterministic, shared, zero per-player cost.

**Files:** `src/content/data/npcs.js` (intent pools), `src/commands/talk.js` or `src/systems/npc-system.js` (selector logic), `src/state/store.js` (`npcVisitCounts`), `src/tests/` (intent selector regression)

## Acceptance Bar

* A `threat >= 3` day triggers a wild event toast styled differently from regular `world:event` toasts
* `market_inversion` observably changes shop prices for the day (verify via shop command)
* Talking to the barkeep during a wild event produces a wild-event-aware line
* Returning to the same NPC 3+ times produces a `repeat_visit` pool line rather than a base line
* `npm run build` passes; `npm test` passes; regression tests cover wild-event derivation and intent selector
