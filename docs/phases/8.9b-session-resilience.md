# Phase 8.9b — Reconnect, Persistence, and Session Resilience

Sourced from the gameplay audit (2026-05-15). Items below are confirmed issues deferred from the 8.78 fix pass.

## Persistence

- **Immediate save on key state changes** — saves are debounced by 5 seconds; a tab crash or hard kill inside that window can lose progress. Fix: keep debounce for noise but force immediate saves on explicitly player-visible state changes (quest completion, level-up, major item transactions).
- **Quest-item protection for deliveries** — `courier_run` uses ordinary `ale` as the delivery item; it is consumable, droppable, and sellable before delivery. Fix: mark quest delivery targets as protected in inventory, or re-grant them via the quest system if lost.
