## P0 — Tab Visibility / Render Cache Bug

**Symptom:** Dark canvas and invisible player after backgrounding tab for a few minutes.

**Root cause:** `spriteCache` (`Map<string, OffscreenCanvas>` in `EntityRenderSystem`) and `tileCache.canvas` (`OffscreenCanvas` in `MapRenderSystem`) survive tab backgrounding as live references, but the browser can zero the underlying bitmap data. Both caches report hits and blit blank canvases on return. The existing `visibilitychange` handler (`loop.js:32`) only resets the timestamp.

**Fix:**
1. Add `invalidate()` to `MapRenderSystem`: sets `this.tileCache = null`
2. Add `invalidate()` to `EntityRenderSystem`: clears `this.spriteCache` Map
3. Wire `visibilitychange` in `src/app/runtime.js` to call both when `document.visibilityState === 'visible'`
4. Optionally: cap `spriteCache` at 64 entries (LRU evict) to prevent unbounded growth

**Files:** `src/systems/map-render-system.js:19`, `src/systems/entity-render-system.js:17`, `src/app/runtime.js`

## P0b — Smooth Player Movement

**Symptom:** Jerky movement despite tween system.

**Two issues:**
1. `TWEEN_SPEED = 10.0` → ~100ms per tile — reads as snap, not glide
2. `movement-system.js:~177` sets `startX/Y` to last canonical position — snaps back on rapid input

**Fixes:**
1. Change `TWEEN_SPEED` from `10.0` to `6.0` in `tween-system.js:24` → ~167ms glide
2. Use current interpolated visual position as tween start:
```js
const prev = this.world.getComponent(entityId, Component.Tweenable);
const vx = prev ? prev.startX + (prev.targetX - prev.startX) * prev.progress : oldX;
const vy = prev ? prev.startY + (prev.targetY - prev.startY) * prev.progress : oldY;
this.world.setComponent(entityId, Component.Tweenable, { startX: vx, startY: vy, targetX: nx, targetY: ny, progress: 0 });
```
3. Set key-repeat interval to `167ms` in `src/engine/input.js` to match tween duration

**Files:** `src/systems/tween-system.js:24`, `src/systems/movement-system.js:~177`, `src/engine/input.js`

## P2 — Weather Visual Effects

**Problem:** Storm and fog drive mechanics and dialogue but canvas looks identical regardless.

**Implementation:**
1. Add `WeatherRenderSystem` in `src/systems/weather-render-system.js`. Insert **after** entity render, **before** UI render in `src/app/runtime.js`.
2. **Storm:** ~40 rain streaks per frame as short diagonal lines (~30°, 3–6px). Positions: `(Date.now() / speed + i * offset) % canvasHeight` — deterministic per frame. Alpha 0.35. Subtle dark blue tint overlay (alpha 0.08).
3. **Fog:** Radial gradient overlay, opaque at edges, transparent at center. Slow sin boundary animation (`Date.now() / 3000`). Alpha 0.4 wilderness/dungeon, 0.2 town. Alternative: reduce entity sprite alpha + CSS `filter: blur(1px)` on canvas element.
4. **Clear/night:** No overlay; optional warm vignette indoors (alpha 0.05).
5. **Transitions:** Lerp overlay alpha over 90 frames when `worldState.weather` changes.
6. `worldState.weather` is already derived deterministically from `seededRNG(hashStr(seed + day + 'daytick'))` in `src/rules/world.js` — visuals are a pure presentation layer, no new persistence. Gate weather overlay to outdoor/wilderness zones only via room `zone` field.

**Files:** `src/systems/weather-render-system.js` (new), `src/app/runtime.js`, `src/state/store.js` (read-only)

## P3 — Animated Sprites: Multi-Frame RLE Format

**Problem:** All sprites are single static frames; no frame cycling.

**RLE format:** each row is an array of `[count, char]` pairs (mirrors OpenRCT2's g1.dat alternating transparent-run/colour-run pairs). Frame-tag naming (`idle`, `walk_n`, `walk_s`, `attack`) follows Aseprite `.ase` frame-tag structure. Both are reference implementations only — no runtime dependency.

**Asset format extension:**
```js
"wolf": {
  shape: [...],       // existing single frame (fallback / frame 0)
  frames: [           // optional: RLE-encoded frames
    [[3,"0"],[2,"1"],[11,"0"],...],  // frame 0
    [[3,"0"],[1,"2"],[1,"1"],...]    // frame 1
  ],
  frameRate: 4        // frames per second
}
```

**Pipeline changes** (`scripts/compile-assets.js`, `scripts/lib/asset-pipeline.js`):
- Detect multi-frame PNGs by `width % height === 0` (e.g. 48×16 = 3 × 16×16 frames)
- Extract each frame, RLE-encode each row, write `frames[]` array

**Runtime decoder:** add `decodeRLEFrame(rleRows)` in `src/graphics/graphics.js` near `applyPalette()` (~line 767). Cache decoded frames on first use.

**Renderer integration** (`src/systems/entity-render-system.js`):
- Add `animFrame` to `Sprite` component
- Each frame: `sprite.animFrame = Math.floor(Date.now() / (1000 / frameRate)) % frameCount`

**Animation sets (minimum viable):**
| Sprite | Frames | Interval | Effect |
|---|---|---|---|
| Player idle | 2 | 800ms | Slight body shift / blink |
| NPC (warm/trade/sage) | 2 | 1200ms | Subtle sway |
| Wolf | 2 | 500ms | Tail wag |
| Enemy idle (wraith, skeleton) | 2 | 700ms | Float/shimmer |
| Torch scenery | 3 | 120ms | Flame flicker |

**Files:** `scripts/compile-assets.js`, `scripts/lib/asset-pipeline.js`, `src/graphics/graphics.js`, `src/systems/entity-render-system.js`, `src/generated/assets/compiled-assets.js` (regenerated)

## P4 — NPC & Enemy Patrol AI

**Problem:** Static entities are visually indistinguishable from scenery.

**Implementation:**
1. Add `Patrol` component (`src/domain/components.js`):
```js
Patrol: { path: [{x,y},...], index: 0, dir: 1, waitTicks: 0 }
```
2. Add `PatrolSystem` (`src/systems/patrol-system.js`), running between `worldSync` and `movement`:
   - Decrement `waitTicks` if > 0, skip
   - Move to `path[index]` via `Tweenable` (reuses tween system)
   - Ping-pong `index` by `dir`; pause 60 ticks at each endpoint
3. Determinism: path position from `(entityId hash + worldState.day + tickCount)` — no `Math.random()`
4. Room definition extension: `staticEntities` entries may include `patrol: [{x,y},...]` — parse in `src/content/define.js:~37`
5. Enemies without authored path: auto-generate 2–3 tile patrol within spawn zone bounding box, seeded by `enemyType + roomId`
6. NPCs with `role: 'static'` skip patrol

**Files:** `src/domain/components.js`, `src/systems/patrol-system.js` (new), `src/app/runtime.js`, `src/content/define.js`, `src/content/data/rooms.js`
